## Why

当前应用主要针对桌面端设计，在移动设备上虽然有基础的响应式处理（768px 断点），但布局压缩严重，用户体验不佳。同时缺少全局缩放功能，无法满足不同用户对界面大小的个性化需求。

## What Changes

- **移动端响应式优化**：改进现有的移动端布局，确保所有模块在小屏幕设备上都能完整展示和操作
  - 优化侧边栏在移动端的展示方式（汉堡菜单或底部导航栏）
  - 调整卡片、按钮、表单控件在小屏幕上的尺寸和间距
  - 优化练耳模块的配置面板和反馈区域的移动端布局
  - 改进和弦图表、指板等可视化组件的移动端适配

- **全局缩放功能**：新增页面缩放控制器，允许用户调整整体界面大小
  - 在侧边栏或顶部添加缩放控制按钮（放大/缩小/重置）
  - 通过 CSS 变量或 transform 实现全局缩放
  - 缩放设置持久化到 localStorage

## Capabilities

### New Capabilities
- `global-zoom-control`: 全局页面缩放功能，包括缩放控制 UI、缩放逻辑实现和状态持久化
- `mobile-responsive-layout`: 移动端响应式布局优化，包括侧边栏适配、模块布局调整和触摸交互优化

## Impact

**受影响的文件**：
- `src/styles/theme.css` - 需要添加缩放相关的 CSS 变量和移动端断点优化
- `src/styles/modules.css` - 需要优化各模块的移动端响应式样式
- `index.html` - 可能需要添加缩放控制 UI 的 HTML 结构
- `src/main.js` - 需要添加缩放控制的 JavaScript 逻辑
- 可能需要新增 `src/utils/zoom-control.js` - 封装缩放功能

**用户体验影响**：
- 移动端用户将获得更好的浏览和操作体验
- 所有用户可以根据个人偏好调整界面大小
- 不影响现有桌面端的使用体验
